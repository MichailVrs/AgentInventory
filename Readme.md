# Система инвентаризации ИТ-активов (Inventory)

Централизованная система инвентаризации ИТ-инфраструктуры предприятия на базе агентов `osquery`.


## Системные требования

Для успешного развертывания и запуска системы на целевом хосте должны быть установлены:
1. **Docker** (версии 20.10 или выше).
2. **Docker Compose** (V2, поддержка синтаксиса `docker compose`).
3. **OpenSSL** (для генерации SSL-сертификатов TLS).
   * *Примечание:* Если утилита `openssl` отсутствует в системе хоста, скрипт инициализации автоматически попытается сгенерировать сертификаты через временный Docker-контейнер Nginx.
4. Доступ к реестрам образов (Docker Hub, Yandex Mirror и Red Soft Registry для скачивания сертифицированных СУБД и Redis).

## Пошаговое развертывание

### Шаг 1. Настройка переменных окружения

Перед первым запуском обязательно скопируйте шаблон конфигурационных файлов `.env.example` в рабочий файл `.env`:

```bash
cp .env.example .env
```

Откройте созданный файл `.env` в текстовом редакторе и настройте параметры:
* **SECRET_KEY:** Сгенерируйте уникальный случайный ключ сессии. Без изменения дефолтного значения сервер выдаст ошибку конфигурации при запуске в рабочей среде.
* **ENROLL_SECRET:** Укажите сложный секретный токен для регистрации агентов osquery.
* **POSTGRESQL_PASSWORD:** Укажите надежный пароль для СУБД PostgreSQL.

---

### Шаг 2. Инициализация системы

Запустите команду инициализации. 

> ⚠️ **ВНИМАНИЕ:** Команда инициализации полностью удаляет все предыдущие тома (volumes) Docker, очищая базу данных и историю логов для чистого старта.

**На Linux / macOS:**
```bash
make init
```

**На Windows (в консоли PowerShell от имени Администратора):**
```powershell
.\init.ps1
```

**Что происходит во время инициализации:**
1. Очистка старых томов и контейнеров (`docker compose down --volumes`).
2. Генерация самоподписанного SSL-сертификата и ключа на 365 дней (`staks/ssl/certs/`).
3. Загрузка базовых и сборка локальных Docker-образов (`inventory-python`, `inventory-osquery` и др.).
4. Применение миграций схемы базы данных к СУБД PostgreSQL.
5. Компиляция и сборка веб-ресурсов (CSS и JS assets) с помощью `manage.py assets build`.

---

### Шаг 3. Запуск системы

После успешной инициализации запустите все контейнеры в фоновом режиме:

```bash
docker compose up -d
```

Проверьте статус запущенных сервисов:

```bash
docker compose ps
```
Все контейнеры (`server`, `worker`, `db`, `proxy`, `broker`, `agent`) должны находиться в статусе `Up` (сервис базы данных должен быть `healthy`).

## Доступ к веб-интерфейсу

Система использует безопасный протокол HTTPS с перенаправлением. Порты опубликованы на хост-системе:

* **Основной адрес панели управления (HTTPS):**  
  **[https://localhost:4444/manage/](https://localhost:4444/manage/)**
* **Автоматическое перенаправление (HTTP):**  
  При переходе на **[http://localhost:8081/manage/](http://localhost:8081/manage/)** прокси-сервер автоматически перенаправит вас на порт `4444` по протоколу HTTPS.

> ⚠️ **Предупреждение о SSL:** Поскольку сгенерированный SSL-сертификат является самоподписанным, ваш браузер отобразит предупреждение о безопасности. Нажмите «Дополнительно» -> «Перейти на сайт (небезопасно)»/«Принять риск и продолжить».

## Проверка готовности (Checklist)

Для контроля работоспособности системы выполните следующие тесты:

1. **Проверка контейнеров:**
   ```bash
   docker compose ps
   ```
   Убедитесь, что все контейнеры запущены.
2. **Запуск интеграционных тестов:**
   Выполните запуск встроенного набора тестов в контейнере сервера:
   ```bash
   docker compose exec server /venv/bin/python -m pytest tests
   ```
   Все тесты должны завершиться со статусом `passed`.
3. **Проверка работы API регистрации (Enroll API):**
   ```bash
   curl -k https://localhost:4444/
   ```
   Должен вернуться пустой ответ со статус-кодом `204` (подтверждение доступности TLS эндпоинта).

## Полезные команды при администрировании

* **Просмотр логов сервера в реальном времени:**
  ```bash
  docker compose logs -f server
  ```
* **Просмотр логов агента osquery:**
  ```bash
  docker compose logs -f agent
  ```
* **Запуск osqueryi для отладки локальных SQL-таблиц:**
  ```bash
  make osqueryi
  ```
* **Остановка всей системы:**
  ```bash
  docker compose down
  ```
* **Полная очистка с удалением всех накопленных данных:**
  ```bash
  docker compose down --volumes --remove-orphans
  ```
