# Чек-лист проверки готовности системы (Smoke Tests & Verification)

Данный документ содержит пошаговый чек-лист для проверки работоспособности развернутой системы инвентаризации ИТ-активов.

---

## 1. Проверка окружения и запуска контейнеров

- [ ] **Конфигурационные файлы:** Файл `.env` существует в корневой директории `AgentInventory/` и заполнен валидными значениями (уникальные секретные ключи, пароли СУБД).
- [ ] **Статус Docker-контейнеров:** 
  При выполнении команды:
  ```bash
  docker compose ps
  ```
  Все 6 контейнеров находятся в состоянии `running` (для PostgreSQL также должно отображаться `healthy`):
  * `inventory-proxy` (Nginx) — порты `0.0.0.0:4444->4443/tcp`, `0.0.0.0:8081->8080/tcp`
  * `inventory-server` (Flask Backend)
  * `inventory-worker` (Celery Tasks)
  * `inventory-broker` (Redis)
  * `inventory-db` (PostgreSQL)
  * `inventory-agent` (Osquery client)

---

## 2. Автоматическое тестирование (Интеграционные тесты)

- [ ] **Запуск pytest в контейнере бэкенда:**
  Выполните команду:
  ```bash
  docker compose exec server /venv/bin/python -m pytest tests
  ```
  *Результат:* Все тесты (включая эндпоинты авторизации, API регистрации и загрузки конфигураций) должны успешно пройти:
  `====== X passed in Y seconds ======`

---

## 3. Доступность веб-интерфейса и API

- [ ] **Работа TLS-портов (HTTPS):**
  Команда:
  ```bash
  curl -kI https://localhost:4444/
  ```
  Возвращает `HTTP/1.1 204 NO CONTENT` (пустой ответ, подтверждающий доступность TLS-сервера).
- [ ] **Редирект с HTTP на HTTPS:**
  Команда:
  ```bash
  curl -I http://localhost:8081/manage/
  ```
  Возвращает статус `HTTP/1.1 301 Moved Permanently` с заголовком редиректа `Location: https://localhost:4444/manage/`.
- [ ] **Отображение панели управления:**
  В браузере при переходе на `https://localhost:4444/manage/` успешно загружается страница веб-интерфейса (без ошибок `502 Bad Gateway` или отсутствующих стилей/скриптов).
- [ ] **Успешная сборка фронтенд-ресурсов:**
  В директории `src/static/public/` сгенерированы и существуют файлы `css/common.css` и `js/common.js`.

---

## 4. Регистрация агента и сбор инвентаризации

- [ ] **Регистрация узла:**
  В веб-интерфейсе в разделе **Узлы (Nodes)** появляется запись о зарегистрированном osquery-клиенте (контейнер `agent` регистрируется автоматически).
- [ ] **Получение конфигурации:**
  Агент запрашивает и получает список плановых SQL-запросов (логи в контейнере `agent` содержат успешные запросы `distributed/read`).
- [ ] **Отправка данных (Snapshot):**
  В разделе информации о зарегистрированном узле в веб-панели отображаются актуальные данные:
  * Информация об ОС и процессоре;
  * Список установленного ПО;
  * Конфигурация сетевых интерфейсов.
