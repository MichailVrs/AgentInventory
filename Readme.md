# AgentInventory

## Быстрый старт

Для первоначальной настройки окружения и развертывания всех необходимых компонентов используйте команду:

```bash
make init
```

Эта команда выполняет полную очистку предыдущих данных, подготовку SSL-сертификатов, сборку/загрузку образов контейнеров и применение миграций базы данных.

### Подробное описание этапов `init`

Команда `init` последовательно запускает следующие задачи:

#### 1. Очистка данных (`data-remove`)
Полностью останавливает запущенные сервисы и удаляет все связанные с ними тома (volumes) и осиротевшие контейнеры. Это гарантирует чистый старт без конфликтов со старыми данными.
*   *Команда:* `docker compose down --remove-orphans --volumes`

#### 2. Генерация SSL-сертификатов (`create-ssl-certificate`)
Создает самоподписанный SSL-сертификат и ключ для безопасного соединения (HTTPS). Сертификат генерируется на 365 дней с использованием конфигурации `staks/ssl/openssl.cnf`.
*   *Результат:*
    *   Ключ: `staks/ssl/certs/certificate.key`
    *   Сертификат: `staks/ssl/certs/certificate.crt`

#### 3. Подготовка образов (`images-build-and-pull`)
Загружает актуальные базовые образы из реестров и собирает локальные образы приложения:
*   **Nginx:** Загружается официальный образ `nginx` и переименовывается в `inventory-nginx`.
*   **PostgreSQL:** Загружается образ `registry.red-soft.ru/ubi8/postgresql-17:17-260414` и переименовывается в `inventory-postgresql`.
*   **Redis:** Загружается образ `registry.red-soft.ru/ubi8/redis:8-260414` и переименовывается в `inventory-redis`.
*   **Python (Server):** Собирается локальный образ `inventory-python` из `docker/python/`.
*   **Osquery (Agent):** Собирается локальный образ `inventory-osquery` из `docker/osquery/`.

> **Примечание:** После тегирования оригинальные загруженные образы удаляются для экономии места.

#### 4. Миграция базы данных (`db-migrate`)
Запускает контейнер сервера для применения всех ожидающих миграций SQLAlchemy/Alembic к базе данных PostgreSQL. После успешного выполнения миграций сервисы останавливаются.
*   *Команда:* `docker compose run --rm server venv/bin/python manage.py db upgrade`

---

### Полезные команды

*   **Запуск osqueryi для отладки:**
    ```bash
    make osqueryi
    ```
    Запускает интерактивную оболочку osquery внутри контейнера агента без подключения к остальным сервисам.

## Доступ к веб-интерфейсу

После выполнения инициализации запустите сервисы в фоновом режиме:

```bash
docker compose up -d
```

Веб-приложение доступно по HTTPS. Откройте браузер и перейдите по адресу:

**[https://localhost/manage/](https://localhost/manage)**

> ⚠️ **Важно:** Так как используется самоподписанный SSL-сертификат, браузер покажет предупреждение о безопасности. Вам необходимо подтвердить исключение безопасности ("Advanced" -> "Proceed to localhost"), чтобы получить доступ к интерфейсу.
